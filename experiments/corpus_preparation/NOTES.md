## 1. Проблема: 176 тысяч файлов — это слишком много

| Аспект | Текущее состояние | Проблема |
|--------|------------------|----------|
| **Количество файлов** | 176,407 | ОС не любит миллионы файлов в одной папке |
| **Размер на диске** | 408 GB | Нормально |
| **Средний размер файла** | ~2.3 MB | Мелкие файлы — много накладных расходов |
| **Чтение/запись** | 176k операций | Медленно |

---

## 2. Оптимальное количество файлов

| Количество файлов | Размер одного файла | Плюсы | Минусы |
|-------------------|---------------------|-------|--------|
| **1,000** | ~400 MB | Удобно для копирования | Один файл может быть проблемой при сбое |
| **10,000** | ~40 MB | Хороший баланс | Всё ещё много файлов |
| **50,000** | ~8 MB | Мелкие файлы, быстрая выборка | Много операций ввода-вывода |
| **100,000** | ~4 MB | Как сейчас | Не оптимально |

### Рекомендация: **10,000 файлов**

| Аргумент | Обоснование |
|----------|-------------|
| **Размер файла** | ~40 MB — удобно для обработки |
| **Количество** | 10k — не слишком много для файловой системы |
| **Параллелизм** | 10k файлов можно обрабатывать 32 воркерами |
| **Восстановление** | При сбое теряется только 40 MB |

---

## 3. Сжимать или нет?

### Вариант А: Без сжатия (JSONL)

| Плюсы | Минусы |
|-------|--------|
| Быстрый доступ | Большой объём (~300-350 GB после очистки) |
| Можно читать построчно | Занимает место |
| Совместимость | — |

### Вариант Б: Со сжатием (JSONL.gz)

| Плюсы | Минусы |
|-------|--------|
| Экономия места (~70-80%) | Медленнее на чтение/запись |
| ~70-100 GB после очистки | Нужно распаковывать на лету |
| Меньше места на диске | Дополнительная нагрузка на CPU |

### Вариант В: Parquet (бинарный, колоночный)

| Плюсы | Минусы |
|-------|--------|
| Очень быстрый | Не человекочитаемый |
| Хорошее сжатие | Требует дополнительных библиотек |
| Интеграция с Spark/Polars | Сложнее отлаживать |

### Моя рекомендация: **JSONL.gz (сжатый)**

| Аргумент | Обоснование |
|----------|-------------|
| **Экономия места** | 400 GB → ~100 GB |
| **Скорость** | Gzip компрессия/декомпрессия быстрая |
| **Совместимость** | Все инструменты умеют читать .gz |
| **Потоковая обработка** | Можно читать построчно через gzip.Reader |

### Плюсы

## 2026-04-16: Стратегия хранения отфильтрованного корпуса

### Решение
- Формат: JSONL.gz (сжатый)
- Количество файлов: 10,000
- Размер файла: ~30-40 MB
- Ожидаемый общий размер: ~80 GB

### Обоснование
- 10k файлов — баланс между количеством и размером
- Gzip сжатие экономит 70-75% места
- Потоковое чтение из .gz без распаковки всего файла
- Совместимость со всеми инструментами

### Параметры
- Уровень сжатия: 1-3 (быстрое)
- Буфер записи: 100k предложений
- Нумерация: part_00000.jsonl.gz ... part_09999.jsonl.gz
