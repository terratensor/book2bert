# План книги "Создание BERT с нуля"

## Структура

### Глава 1: Зачем создавать BERT с нуля?
- Учебная цель: разобрать архитектуру "до атомов"
- Практическая цель: получить модель для уникального корпуса
- Почему нельзя взять готовую модель?

### Глава 2: Подготовка данных — самый важный этап
- Исходные данные: 180k книг, 40 ГБ
- Сегментация предложений (razdel + HTTP-сервис)
- Проблема: списки и таблицы → решение: разбивка по `\n`
- Проблема: CJK и тайские символы → фильтрация

### Глава 3: Токенизация — мост между текстом и моделью
- Что такое WordPiece и зачем он нужен
- Проблема: словарь на 50 книгах не содержит целых слов
- Решение: обучение токенизатора на 4 ГБ корпуса
- Ускорение: Go-утилита вместо Python

### Глава 4: Сборка датасета для BERT
- Формат JSONL для BERT
- Проблема: двойной `[CLS]` и `[SEP]`
- Проблема: потеря `[SEP]` между предложениями
- Ускорение: батчевая токенизация (17 минут вместо 6-8 часов)

### Глава 5: Архитектура BERT (реализация на PyTorch)
- Multi-Head Self-Attention
- Positional Encoding
- Transformer Block
- BERTForMLM

### Глава 6: Обучение
- Streaming-загрузка данных
- Mixed precision (fp16)
- Стратегия чекпоинтов
- OneCycleLR scheduler
- Возобновление обучения после сбоев

### Глава 7: Результаты
- Эпоха 1: val_loss 3.43, perplexity 30.8
- Эпоха 2: val_loss 2.96, perplexity 19.3
- MLM предсказания: от знаков препинания к осмысленным словам

### Глава 8: Выводы и уроки
- Качество данных важнее архитектуры
- Токенизатор на большом корпусе — ключевой фактор
- Python + Go: синергия для больших данных
- Что мы сделали неправильно и как исправили
