# Приложение В: Словарь терминов

## A
- **Attention** — механизм, позволяющий модели "смотреть" на разные части входной последовательности
- **Attention mask** — маска, указывающая, какие токены реальные, а какие — padding

## B
- **BERT** — Bidirectional Encoder Representations from Transformers
- **BPE** — Byte-Pair Encoding, алгоритм токенизации

## C
- **CJK** — китайские, японские и корейские символы
- **CLS** — специальный токен в начале последовательности, его эмбеддинг используется для классификации

## E
- **Embedding** — векторное представление токена
- **Epoch** — один полный проход по всему обучающему датасету

## F
- **FFN** — Feed-Forward Network, часть трансформера
- **Fine-tuning** — дообучение предобученной модели на конкретную задачу

## G
- **GELU** — Gaussian Error Linear Unit, функция активации
- **Gradient accumulation** — техника для эмуляции большого batch size при ограниченной памяти
- **Gradient clipping** — обрезание градиентов для стабилизации обучения

## H
- **Hidden size** — размерность эмбеддингов в модели
- **HTTP сервис** — способ коммуникации между Go и Python

## I
- **Input IDs** — числовые идентификаторы токенов
- **Intermediate size** — размер внутреннего слоя FFN (обычно 4 × hidden_size)

## J
- **JSONL** — формат хранения данных (каждая строка — JSON объект)

## L
- **Layer norm** — нормализация по признакам (stabilizes training)
- **Learning rate** — скорость обучения
- **Loss** — функция ошибки (в нашем случае cross-entropy)

## M
- **Masked LM (MLM)** — задача предсказания маскированных токенов
- **Mixed precision (fp16)** — использование 16-битных чисел для ускорения и экономии памяти
- **Multi-head attention** — несколько параллельных голов внимания

## N
- **Next Sentence Prediction (NSP)** — задача определения, следует ли B за A
- **Num heads** — количество голов внимания
- **Num layers** — количество трансформерных блоков

## O
- **OneCycleLR** — scheduler с разогревом и снижением LR

## P
- **Padding** — добавление специальных токенов для выравнивания длины
- **Perplexity** — экспонента от loss, метрика качества языковой модели
- **Positional encoding** — добавление информации о позиции токена

## Q
- **Q, K, V** — Query, Key, Value — матрицы в механизме внимания

## R
- **Residual connection** — `x + f(x)`, помогает градиентам проходить сквозь слои

## S
- **Scheduler** — планировщик learning rate
- **SEP** — специальный токен для разделения предложений
- **Streaming** — чтение данных по частям, без загрузки в память

## T
- **Tokenization** — превращение текста в токены
- **Token type IDs** — идентификаторы сегментов (для NSP)
- **Transformer** — архитектура на основе внимания
- **t-SNE** — метод визуализации многомерных данных

## W
- **WordPiece** — алгоритм токенизации, используемый в BERT
- **WandB** — инструмент для визуализации экспериментов

## Дополнительно
- **[PAD]** — токен для выравнивания длины
- **[UNK]** — токен для неизвестных слов
- **##** — префикс, обозначающий продолжение предыдущего токена
