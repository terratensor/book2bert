# Глава 1: Зачем создавать BERT с нуля?

## 1.1 Мотивация

В современном мире NLP у нас есть замечательная библиотека `transformers` от Hugging Face. Мы можем написать:

```python
from transformers import BertForMaskedLM, BertTokenizer
model = BertForMaskedLM.from_pretrained('bert-base-uncased')
```

И получить готовую модель, обученную на миллиардах слов. Зачем же создавать BERT с нуля?

**Две цели:**

1. **Учебная:** Разобрать архитектуру "до атомов". Понять, как работает Multi-Head Attention, почему Positional Encoding важен, что происходит внутри Transformer Block.

2. **Практическая:** Получить модель для уникального корпуса, где готовые модели не работают.

## 1.2 Наш корпус

У нас есть библиотека из **180,000 книг** (40 ГБ в сжатом виде, ~200 ГБ текста) с преимущественно нехудожественной литературой.

**В этой книге** мы фокусируемся на военно-историческом подразделе — **militera**:

| Показатель | Значение |
|------------|----------|
| Исходные файлы (сжатые .txt.gz) | **3.6 ГБ** |
| Распакованный текст | ~15-20 ГБ |
| Корпус после фильтрации (corpus.txt) | **14 ГБ** |
| Количество предложений | **93,124,036** |
| Количество файлов | 11,361 |

Жанровое разнообразие полной библиотеки включает:

| Жанр | Фрагментов (~2200 символов) |
|------|---------------------------|
| История | 8,201,168 |
| Биографии и мемуары | 4,314,080 |
| **Милитера** | **3,675,708** |
| Публицистика | 2,232,364 |
| ... и еще 50+ жанров | ... |

После успешного завершения эксперимента на militera, мы масштабируем подход на всю библиотеку (40 ГБ сжатых данных → ~200 ГБ текста).

## 1.3 Особенности militera корпуса

| Особенность | Пример | Влияние |
|-------------|--------|---------|
| **Военная терминология** | "шкураки", "бруствер", "плацдарм" | Готовые модели не знают эти слова |
| **OCR-артефакты** | "внима тельно" (разрыв слов) | Нужна предобработка |
| **Старая орфография** | "ъ", "ѣ", "i" | Частично присутствует |
| **Списки без точек** | Списки фамилий, библиография | Сегментатор не разбивает |
| **Длинные предложения** | Юридические тексты, описания | Могут превышать 512 токенов |

## 1.4 Почему нельзя взять готовую модель?

| Проблема | Готовая модель | Наш подход |
|----------|----------------|------------|
| Военная терминология | `[UNK]` | Учим токенизатор на корпусе |
| OCR-артефакты | Ломается | Чистим и нормализуем |
| Старая орфография | Не понимает | Выучит из данных |
| Списки без точек | Не разбивает | Разбиваем по `\n` |

## 1.5 Что мы хотим получить

1. **Понимание** архитектуры трансформеров
2. **Рабочий пайплайн** от текста до модели
3. **Модель**, которая понимает специфику военно-исторических текстов
4. **Открытый код** и **документация** для воспроизведения
