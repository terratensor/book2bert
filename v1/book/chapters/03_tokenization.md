# Глава 3: Токенизация — мост между текстом и моделью

> "Слова — это атомы смысла. Токенизатор решает, что считать атомом."

## 3.1 Что такое токенизация и зачем она нужна

Токенизация — это процесс превращения текста в последовательность чисел (токенов), которые может обработать нейронная сеть.

**Проблема:** Модель не понимает буквы и слова. Она понимает только числа.

**Решение:** Каждому токену (слову, части слова, знаку препинания) присваивается уникальный идентификатор (ID).

```
Текст: "Философия есть учение"
   ↓ токенизация
Токены: ["Философия", "есть", "учение"]
   ↓ словарь
ID: [7659, 757, 5010]
```

## 3.2 Почему нельзя просто взять словарь всех слов?

| Проблема | Пример | Последствия |
|----------|--------|-------------|
| Размер словаря | Русский язык ~500,000 слов | Модель будет огромной |
| Редкие слова | "шкураки" (военный жаргон) | Не войдут в словарь → `[UNK]` |
| OCR-ошибки | "внима тельно" | Неизвестные последовательности |
| Старая орфография | "ъ", "ѣ" | Требуют отдельных токенов |

**Решение:** WordPiece — алгоритм, который разбивает слова на **субворды** (части слов).

### 3.3 Почему WordPiece, а не BPE?

**BPE (Byte-Pair Encoding)** сливает самые частотные пары символов. 
**WordPiece** выбирает слияния, максимизирующие вероятность данных.

Ключевые отличия:
- WordPiece использует маркер `##` для субвордов (не могут начинать слово)
- WordPiece лучше выделяет морфемы (корни, суффиксы, окончания)
- WordPiece обеспечивает однозначную обратимость токенизации

Для русского языка, богатого морфемами, WordPiece предпочтительнее.
Он позволяет модели понимать грамматику через окончания (`##я`, `##е`, `##м`).

## 3.4 Как работает WordPiece

### Принцип

WordPiece начинается со всех символов и наиболее частотных слов, затем итеративно **сливает** наиболее частотные пары токенов.

**Пример разбиения:**

```python
# Слово "Философия" может быть разбито по-разному:

# Плохой токенизатор (мало данных):
"Философия" → ["Философ", "##и", "##я"]

# Хороший токенизатор (много данных):
"Философия" → ["Философия"]  # целое слово в словаре
```

**Важно:** Символ `##` означает, что токен является продолжением предыдущего (не может стоять в начале слова).

### Алгоритм

1. **Инициализация:** Словарь = все символы + самые частые слова
2. **Повторять N раз** (где N = vocab_size - начальный размер):
   - Найти самую частотную пару токенов в корпусе
   - Слить их в один новый токен
   - Добавить в словарь

**Пример слияния:**
```
"Философ" + "и" → "Философи"
"Философи" + "я" → "Философия"
```

## 3.5 Проблема: словарь на 50 книгах не содержит целых слов

В нашем первом эксперименте мы обучили токенизатор на **50 философских книгах** (27 МБ текста).

**Результат:**

```python
# Токенизация слова "невзгод"
"невзгод" → ["нев", "##з", "##го", "##д"]  # бессмысленные субворды!
```

**Почему так произошло?**
- Корпус слишком мал (50 книг, 152k предложений)
- Редкие слова не встречались достаточно часто
- Алгоритм не смог выучить осмысленные морфемы

**Последствия для модели:**
- Модель училась предсказывать окончания слов (`##я`, `##е`), а не смысл
- MLM предсказывала знаки препинания вместо осмысленных слов
- Эксперимент провалился

## 3.6 Решение: обучаем токенизатор на militera корпусе

**Данные для обучения:**
- 11,361 книга (3.6 ГБ сжатых, 14 ГБ текста)
- 93,124,036 предложений
- Военная терминология, специфическая лексика

**Результат после обучения с vocab_size=50000:**

```python
# Токенизация тех же слов на новом токенизаторе
"невзгод" → ["невз", "##год"]     # осмысленные части!
"философия" → ["философия"]        # целое слово!
"методология" → ["метод", "##ология"]  # корень + суффикс!
```

**Сравнение:**

| Слово | Токенизатор на 50 книгах | Токенизатор на militera |
|-------|-------------------------|------------------------|
| невзгод | `['нев', '##з', '##го', '##д']` | `['невз', '##год']` |
| философия | `['философи', '##я']` | `['философия']` |
| методология | `['метод', '##ология']` | `['метод', '##ология']` |

## 3.7 Сбор корпуса для токенизатора

**Проблема:** Сбор корпуса на Python (чтение 93M предложений, фильтрация CJK, запись в файл) занимал **10-15 минут**.

**Решение:** Переписали на Go.

```bash
go run cmd/corpus-builder/main.go \
    --sentences data/processed/sentences_militera_v3 \
    --output data/processed/corpus_militera_v3.txt \
    --workers 16 \
    --filter-cjk true
```

**Результат:**
```
2026/04/04 02:57:58 Found 11351 JSONL files
2026/04/04 02:58:57 Done! Total sentences: 93124036
2026/04/04 02:58:57 Corpus saved to: data/processed/corpus_militera_v3.txt
```

**Время: 59 секунд** (вместо 10-15 минут на Python).

## 3.8 Обучение токенизатора

```bash
python scripts/train_tokenizer_from_corpus.py \
    --corpus data/processed/corpus_militera_v3.txt \
    --output-dir data/processed/tokenizer_militera_v3 \
    --vocab-size 50000
```

**Процесс обучения:**

## 3.10 Анализ длинных предложений

Перед сборкой датасета мы проанализировали, сколько предложений превышают лимит в 512 токенов.

```bash
python scripts/analyze_long_sentences.py \
    --sentences-dir data/processed/sentences_militera_v3 \
    --tokenizer-path data/processed/tokenizer_militera_v3 \
    --min-tokens 512 \
    --batch-size 2000
```

**Результат:**
```

Sentences >=512 tokens: 770
Percentage: 0.0008%
```

**Вывод:** Всего 770 длинных предложений из 93 миллионов (0.0008%) — их можно безопасно обрезать до 512 токенов без потери качества.
